浅析Java中的final修饰符

Posted on 2018-06-12 | In Java | reading

　　关于Java中的final关键字一直以为自己已经掌握了，今天复习时才发现之前了解的并不全面，所以将一些东西记录下来，方便以后翻看。

  1. final修饰类
    当final关键字用来修饰一个类时，这个类将不能被继承。而final类中的成员变量不会受影响，可以根据需要将成员变量设置为final。
    另外final类中的成员方法会被隐式的指定为final方法。这是我之前没有了解的。
    在使用final修饰类的时候，还是应该注意谨慎选择，除非确定这个类真的在以后不会用来继承或者出于安全的考虑，尽量不要将类设计为final类。
  2. final修饰方法
    在使用final修饰方法时，一般是出于禁止该方法在以后被重写，即在子类中被覆盖(Override)。早期Java版本中，final方法会被转化为内嵌调用，但当方法过于庞大时
    可能看不到这种转化带来的性能提升，现在的Java版本中已经不需要使用final方法进行这些优化了。
    另外，类的private方法会被隐式的指定为final方法
  3. final修饰变量
    当final修饰一个变量时，首先如果该变量是一个原始类型，则其值一旦指定便不可更改。如果是引用类型的变量，则在指定引用后
    便不可以让其指向另一个变量。
  • 1 当final修饰类中的成员变量时，成员变量必须在定义时或构造器中被赋值(局部变量只需在使用前赋值即可)。

  • 2 要注意final修饰的引用型变量是可以改变其值的
    被final修饰的引用型变量可以在运行期间改变其值： 

    class MyTest {

  public MyTest() {
  }

  public final List<Integer> alist = new ArrayList<>();

  public static void main(String[] args) {
      MyTest subTest = new MyTest();
      List<Integer> mylist = subTest.alist;
      mylist.add(1);
      System.out.println(mylist);
  }
}
